Data Engineering & Big Data Architecture

Dataset Scale & Storage Strategy

The system is powered by the Gaia DR3 dataset (originally ~1.8 billion sources), filtered to approximately 557 million high-confidence stars. Data is stored in Google BigQuery using a columnar, serverless architecture optimized for analytical workloads.

Tables are partitioned and clustered to minimize scan size and cost. Spatial filtering is performed using Healpix indexing, and range queries are optimized for distance-based selection (light years). This design enables sub-second queries over gigabyte-scale datasets without infrastructure management.

Query Optimization & Cost Efficiency

Queries are dynamically generated and parameterized to ensure security and performance. Only required columns are selected, full table scans are avoided, and filtering is applied before projection calculations. The architecture is designed to remain within BigQueryâ€™s free query tier whenever possible while maintaining predictable per-query cost.

Proper Motion & Kinematic Projection Engine

To provide physically meaningful visualization, the backend implements a simplified stellar kinematic model. Right ascension, declination, and parallax values are converted to Cartesian coordinates.
